# Keyboard Shortcuts

Quick reference for every keyboard and mouse interaction in DocCrate.

---

## Scrolling

| Key / Input | Action |
|-------------|--------|
| `↓` Down Arrow | Scroll down one line (~21 px) |
| `↑` Up Arrow | Scroll up one line (~21 px) |
| `Page Down` | Scroll down 85% of the window height |
| `Page Up` | Scroll up 85% of the window height |
| `Home` | Jump to the very top of the document |
| `End` | Jump to the very bottom of the document |
| Mouse wheel | Smooth continuous scroll (120 units per notch) |
| Drag scrollbar thumb | Drag the rounded thumb on the right edge |

---

## Document Navigation

| Key / Input | Action |
|-------------|--------|
| `←` Left Arrow | Go back to the previous document in history |
| `→` Right Arrow | Go forward in history |
| Click sidebar item | Navigate to that document (pushes history) |
| Click local link | Navigate to the linked document (pushes history) |
| Click external link | Open the URL in your default browser |

> Navigation with `←` and `→` works identically to browser back/forward.
> Every sidebar click and every local link click pushes an entry.
> Navigating to a new page clears the forward stack.

---

## Sidebar

| Key / Input | Action |
|-------------|--------|
| `Ctrl+B` | Toggle sidebar open / closed |
| Click `‹` chevron | Collapse sidebar to reveal tab |
| Click `›` chevron | Restore sidebar to last width |
| Click reveal tab | Restore sidebar (same as `›`) |
| Drag divider | Resize sidebar width (min 80 px, max 65% of window) |

---

## Window

| Key / Input | Action |
|-------------|--------|
| Drag title bar | Move window |
| Drag window edge | Resize window — content reflows automatically |
| Minimum window size | 320 × 240 px |

---

## Mouse Cursors

DocCrate changes the cursor to give feedback about what will happen on click:

| Cursor | When it appears |
|--------|----------------|
| Arrow (`↖`) | Default — over content, scrollbar track, inactive area |
| Hand (`☞`) | Over a hyperlink, the toggle button, or the reveal tab |
| Resize horizontal (`↔`) | Over the sidebar/content divider |

---

## Tips

- There is no menu bar or toolbar — everything is reachable from the keyboard
  or a single mouse click
- `Ctrl+B` is the fastest way to give the content area more horizontal space
- `Home` / `End` are useful for long reference pages like [Architecture](architecture.md)
- `←` / `→` let you treat DocCrate like a browser; follow links freely and
  navigate back without reaching for the sidebar
